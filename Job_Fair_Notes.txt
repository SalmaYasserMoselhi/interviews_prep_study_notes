* العبادات اولا
* ادعي كتير واستعين بالله، ورزقك مكتوب.
* اقرا الحديث ده : https://dorar.net/hadith/sharh/36167
* التوتر والخوف طبيعي جدًا المهم تحول الخوف ده لطاقه تحركك
* فترة التحضير دي فرصة حلوة تتعلم فيها حاجات مكانش عندك وقت تفهمها قبل كده.
* لو مجالكش انترفيو أو جالك عدد قليل، متزعلش، محدش عارف الخير فين.
* قدم في أي شركة مناسبة، متستناش شركات الـ ITI بس.

---

# قبل أي انترفيو

اعرف الشركة كويس.

لما تعرف اسم الشركة أو الـ Interviewer (وده غالبًا بيكون قبلها بيومين تقريبًا):

* اعرف الشركة شغالة في إيه.
* اعرف الـ Tech Stack.
* اعرف الفريق اللي هتشتغل معاه.
* شوف محتاجين إيه.
* لو تعرف حد شغال هناك، اسأله عن:

  * شكل الانترفيو.
  * بيسألوا في إيه.
  * إيه اللي يركزوا عليه.
  * بيئة الشغل عاملة إزاي.

جهز لكل شركة لوحدها لو قدرت.

---

# ذاكر الأساسيات الأول

ابدأ بـ:

* أساسيات اللغة.
* OOP.
* SOLID.
* DSA.
* Database (sql,indexes,views,..).

بعدها ابقى ذاكر المواضيع الكبيرة.

ركز على اللغة اللي مقدم بيها.

* ذاكر أشهر Interview Questions.
* ذاكر Code Snippets.
* راجع أهم الـ Concepts.
* و اعمل كده مع الباقي و هما Java - Python - JavaScript - TypeScript - PHP - Ruby.

---

# Design Patterns

راجع أشهر الـ Design Patterns.

مش تحفظها، لكن اعرف:

* ليه بنستخدمها؟
* بتحل إيه؟
* استخدمتها فين؟
* ولو مستخدمتهاش، كانت هتفيد فين؟

---

# خد فكرة عن المواضيع دي

مش لازم تبقى Expert، لكن ابقى فاهم الفكرة.

* Microservices
* gRPC
* Message Queues
* Caching
* Load Balancers
* Rate Limiting
* Throttling
* Monolithic vs Microservices
* أساسيات System Design
* DDD

---

# Security

راجع أشهر هجمات OWASP.

خصوصًا:

* XSS
* SQL Injection
* DDoS

وغيرهم و اعرف أمثلة عليهم وإزاي تمنعهم.

---

# راجع مشاريعك

دي من أكتر الحاجات اللي بيتسأل فيها.

اعرف تشرح:

* المشروع بيعمل إيه.
* الـ Architecture.
* ليه اخترت الـ Design ده.
* المشاكل اللي قابلتك.
* إزاي حليتها.
* وإيه اللي ممكن تحسنه.

واتدرب ترسم Architecture بسيطة للمشروع و مشاريع مختلفه.

اعرف ازاي ممكن تحسن Performance لمشروع بتاعك او اي مشروع في العموم (db, indexes,load balancers, microservices, caching, etc)

---

# Problem Solving

فكر خطوة بخطوة.

متستعجلش الإجابة.

خلي الـ Interviewer يشوف طريقة تفكيرك.

---

# في الانترفيو

* اتكلم بثقة.
* لو متعرفش حاجة، قول "مش عارف" عادي.
* وبعدها فكر بصوت عالي.
* خد الـ Hints وابني عليها.
* جاوب بأمثلة من مشاريعك أو من الحياة.
* وريه إنك فاهم مش حافظ.
* بين إنك بتحب تتعلم.
* الحفركه حلوه مفيش كلام

---

# الـ Behavioral مهم

ناس كتير بتعدي الـ Technical وتقع في الـ Behavioral أو الـ Situational.

راجع أسئلة زي:

* عرف نفسك.
* ليه الشركة دي؟
* احكي عن Challenge حصل معاك.
* اشتغلت مع Team إزاي؟
* نقطة ضعفك.
* اتعاملت مع مشكلة إزاي.

وخلي كلامك يبين إنك:

* بتحب تتعلم.
* بتعرف تشتغل مع Team.
* بتتقبل الـ Feedback.
